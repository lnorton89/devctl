## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User click to client API | UI actions trigger process control requests. |
| Client polling to backend | Browser receives lifecycle status and log-tail data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Repudiation | lifecycle UI actions | accept | Phase 2 has no audit log; visible state changes and run history cover local single-user operation. |
| T-02-04-02 | Denial of Service | status polling | mitigate | Poll only projects acted on, stop at terminal states, and stop after three consecutive polling errors. |
| T-02-04-03 | Information Disclosure | status/log tail display | accept | Status and log tail are intended local single-user UI data. |
</threat_model>
