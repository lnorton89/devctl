<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to lifecycle API | User actions trigger process lifecycle endpoints. |
| Lifecycle API to registry | Route handlers load persisted project config. |
| Lifecycle API to parser and process manager | Stored path/scriptName are validated before execution. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering | start/restart route scriptName | mitigate | Validate `project.scriptName` against `parsePackageJson(project.hostPath).scripts` immediately before processManager.start/restart. |
| T-02-03-02 | Spoofing | project id route param | accept | v1 is trusted single-user local software; missing project ids return 404. |
| T-02-03-03 | Information Disclosure | parse-scripts route | mitigate | Route returns package scripts only and maps parser errors to controlled messages. |
| T-02-03-04 | Denial of Service | lifecycle route misuse | mitigate | Process manager duplicate-start guard remains the authority; routes do not create separate managers per request. |
</threat_model>
