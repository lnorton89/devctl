<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ProcessManager to child_process.spawn | Validated project script and cwd become OS process inputs. |
| Managed process to stdout/stderr capture | Untrusted process output enters server memory. |
| ProcessManager to OS process tree | Stop/restart sends termination signals or taskkill commands. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Tampering | process command construction | mitigate | Plan 03 validates scriptName against package.json scripts before calling start/restart; manager accepts already-validated scriptName only. |
| T-02-02-02 | Denial of Service | duplicate process spawning | mitigate | start returns existing status for starting, running, and stopping states. |
| T-02-02-03 | Denial of Service | log capture | mitigate | Ring buffer count cap, 4096-character line cap, and five-run history cap bound memory. |
| T-02-02-04 | Tampering | process tree stop | mitigate | D-11 graceful termination escalates only after timeout; process group/tree kill targets only the managed pid. |
| T-02-02-05 | Information Disclosure | logs in memory | accept | Logs are intentionally visible in this trusted single-user local tool and are not persisted in Phase 2. |
</threat_model>
