<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User form input -> React state | Sensitive env values and command strings enter browser state. |
| React client -> Express API | Registry mutations cross into backend validation. |
| API response -> registry table | Persisted data is displayed and must avoid leaking env values in table views. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Information Disclosure | `ProjectTable.tsx` / `ProjectMobileList.tsx` | mitigate | Never render env variable values outside `ProjectFormDrawer`. |
| T-01-03-02 | Tampering | `ProjectFormDrawer.tsx` | mitigate | Validate form data with shared schema before API mutation and display backend validation issues. |
| T-01-03-03 | Elevation of Privilege | `ProjectRegistryPage.tsx` | mitigate | Do not render lifecycle execution controls in Phase 1. |
| T-01-03-04 | Spoofing | UI actions | accept | v1 is trusted single-user local software; no authentication is in scope. |
| T-01-03-05 | Repudiation | Delete action | mitigate | Require explicit destructive confirmation before deleting a registry entry. |
</threat_model>
