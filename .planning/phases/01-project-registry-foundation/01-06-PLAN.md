<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User form input -> React state | Sensitive env values and command strings enter browser state. |
| React client -> Express API | Registry mutations cross into backend validation. |
| API response -> form errors | Backend validation issues become visible helper/error text. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06-01 | Information Disclosure | `ProjectFormDrawer.tsx` | mitigate | Env values render only in the drawer/dialog form and are not copied to table/list or generic error output. |
| T-01-06-02 | Tampering | `ProjectFormDrawer.tsx` | mitigate | Validate form data with shared schema before API mutation and display backend validation issues. |
| T-01-06-03 | Elevation of Privilege | `ProjectRegistryPage.tsx` | mitigate | Do not render lifecycle execution controls in Phase 1. |
| T-01-06-04 | Repudiation | `DeleteProjectDialog.tsx` | mitigate | Require explicit destructive confirmation before deleting a registry entry. |
| T-01-06-05 | Spoofing | UI actions | accept | v1 is trusted single-user local software; no authentication is in scope. |
</threat_model>
