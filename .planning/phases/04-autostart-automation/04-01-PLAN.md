  <action>
    Create `src/server/autostart/autostart.ts` with the following structure:

    **Imports:**
    - `RegistryRepository` type from `../registry/registryRepository.js`
    - `ProcessManager` type from `../process/processManager.js`
    - `ProjectConfig` type from `../../shared/projectSchema.js`

    **Types (exported):**
    - `AutostartError` interface: `{ projectId: string; projectName: string; error: string }`
    - `AutostartResult` interface: `{ started: number; failed: number; errors: AutostartError[] }`

    **Function `autostartProjects(repository, processManager)`:**

    1. Call `repository.listProjects()` to get all projects
    2. Filter to projects where `project.autostart === true`
    3. If no autostart-enabled projects, return `{ started: 0, failed: 0, errors: [] }`
    4. Map each autostart project to a start promise:
       - Validate that `project.scriptName` and `project.hostPath` are both non-empty strings
       - If validation fails, return a rejected promise with a descriptive message
       - Otherwise, call `processManager.start(project.id, project.scriptName, project.hostPath)`
       - Wrap the call in `async () => { try { ... } catch (e) { ... } }` so individual failures don't propagate
    5. Use `Promise.allSettled(startPromises)` to execute all starts in parallel (per parallel startup decision)
    6. Iterate over results:
       - `fulfilled` status entries count as started
       - `rejected` status entries count as failed, collect into `AutostartError[]` with project id, name, and error message
    7. Return `AutostartResult` with started/failed counts and error details

    **Important behavioral rules (from CONTEXT.md decisions):**
    - Parallel startup: `Promise.allSettled`, NOT sequential
    - No retry on failure: individual catch blocks log error content and continue â€” do NOT retry
    - If `start()` throws (occupied port, missing script, ENOENT), capture the error message in the result â€” do not set special state
    - Validate `scriptName` and `hostPath` exist before calling `processManager.start()`
    - Never import from `express` or any route layer â€” this is a pure internal module
    - Function is synchronous in structure but returns a Promise (due to `repository.listProjects()` being async)
    - Do NOT import `createProcessManager` or `createRegistryRepository` â€” they are injected

    **Exports:**
    - Named export `autostartProjects`
    - Named exports for `AutostartResult` and `AutostartError` types
  </action>

    - Returns typed `AutostartResult` with started/failed counts and error details
