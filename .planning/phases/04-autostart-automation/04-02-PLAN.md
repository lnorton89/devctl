    Create a new `useCallback` following the lifecycle handler pattern (similar to `handleStartProject`):

    ```
    const handleToggleAutostart = useCallback(
      async (project: ProjectConfig, autostart: boolean) => {
        // 1. Optimistic update: immediately update local projects array
        setProjects((prev) =>
          prev.map((p) =>
            p.id === project.id ? { ...p, autostart } : p,
          ),
        );

        // 2. Clear previous lifecycle error
        setLifecycleError(null);

        // 3. Call API
        try {
          const updated = await updateProject(project.id, {
            ...project,
            autostart,
          });
          // Use server response to confirm state
          setProjects((prev) =>
            prev.map((p) =>
              p.id === updated.id ? updated : p,
            ),
          );
        } catch (err: unknown) {
          // 4. Rollback optimistic update on failure
          setProjects((prev) =>
            prev.map((p) =>
              p.id === project.id ? { ...p, autostart: !autostart } : p,
            ),
          );
          const message =
            err instanceof Error ? err.message : 'Unknown error';
          setLifecycleError(
            `Could not update autostart for ${project.name}. ${message}`,
          );
        }
      },
      [],
    );
    ```

    **Update render — pass `onToggleAutostart` to both ProjectTable and ProjectMobileList:**
    In the `<ProjectTable>` element (around line 446-458), add: `onToggleAutostart={handleToggleAutostart}`
    In the `<ProjectMobileList>` element (around line 432-444), add: `onToggleAutostart={handleToggleAutostart}`

    **Important rules:**
    - Use `setProjects` with a callback function (`prev => ...`), not with the current state variable, to avoid stale closures
    - The rollback toggles `autostart` back to `!autostart` (its previous value)
    - Show the error via existing `setLifecycleError` — the Snackbar at the bottom of the page already renders lifecycle errors
    - Do NOT await the optimistic update — it's immediate via `setProjects`
    - The `updateProject` call sends the ENTIRE project object (existing pattern from the API), not just `{ autostart }`. Use spread: `await updateProject(project.id, { ...project, autostart })`
    - No loading state or spinner needed — Switch toggles instantly (optimistic) and reverts on error
    - No type annotation changes needed — `handleToggleAutostart`'s signature is derived from usage
